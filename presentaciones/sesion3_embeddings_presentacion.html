🤔 ¿Cómo Entienden las Computadoras el Lenguaje?

El desafío:

  • Computadoras solo procesan números (0s y 1s)
  • El texto es inherentemente NO estructurado
  • Necesitamos traducir: Lenguaje → Números

Ejemplo:

"El gato come pescado"
         ↓
    [¿números?]
         ↓
Computadora puede procesarlo ✓

Bag-of-Words (BoW)

Idea: Representar texto contando palabras

Ejemplo:

Documento 1: "El gato come pescado"
Documento 2: "El perro come carne"

Vocabulario: [el, gato, perro, come, pescado, carne]

Vectores resultantes:
Doc 1: [1, 1, 0, 1, 1, 0]
Doc 2: [1, 0, 1, 1, 0, 1]

✅ Ventajas:

  • Simple de implementar
  • Interpretable
  • Funciona para tareas básicas
📊 Figura 1-5: Bag-of-words illustration

👨‍💻 DEMO: Bag-of-Words en Python

from sklearn.feature_extraction.text import CountVectorizer

textos = [
    "El gato come pescado",
    "El perro come carne",  
    "El gato y el perro son amigos"
]

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(textos)

print("Vocabulario:", vectorizer.get_feature_names_out())
print("Vectores:", X.toarray())

❌ Problemas Fundamentales

1️⃣ Ignora el ORDEN

"El gato persigue al ratón"
"El ratón persigue al gato"
→ ¡Misma representación!

2️⃣ No captura SIGNIFICADO

"gato" ≠ "felino"
(aunque significan lo mismo)

3️⃣ Alta DIMENSIONALIDAD

Vocabulario: 50,000 palabras
→ Vector: 50,000 dimensiones
→ Casi todo son ceros (sparse)
🤔 ¿Cómo capturamos que "gato" y "felino" son similares?

💡 Embeddings: La Solución

Definición:

Vectores densos que capturan el SIGNIFICADO de palabras

Comparación:

Bag-of-Words Embeddings
"¿Está presente?" "¿Qué significa?"
Sparse (muchos 0s) Denso (todos valores)
Alta dimensión Baja dimensión (300)
Ignora significado Captura significado

Ejemplo:

"gato" → [0.2, 0.8, -0.3, 0.5, ...]
         ↑
    300 valores que capturan su esencia

Word2Vec: Aprendiendo Significado

💬 Idea Central:
"Conocerás una palabra por las compañías que tiene"
— John Rupert Firth

Ejemplo:

Contextos donde aparece "gato":

  • "El ___ come pescado"
  • "El ___ maúlla"
  • "El ___ tiene bigotes"
  • "El ___ y el perro juegan"

→ La red neuronal aprende qué significa "gato"
   por las palabras que aparecen cerca

Resultado: Embedding de "gato"

¿Cómo se Entrena Word2Vec?

Tarea: Predecir si dos palabras son VECINAS

Entrada: Par de palabras

"gato" + "pescado"  →  ¿Vecinos? ✓
"gato" + "mesa"     →  ¿Vecinos? ✗

Durante entrenamiento:

  1. Red neuronal hace predicción
  2. Compara con respuesta correcta
  3. Ajusta embeddings para mejorar
  4. Repite millones de veces
Resultado:
→ Palabras con contextos similares
→ Embeddings similares

¿Qué Representan las Dimensiones?

Embedding de "bebé":

[0.1, 0.9, 0.8, 0.2, ...]
  ↑    ↑    ↑    ↑
(ejemplos simplificados)
Comida Humano Joven Alegre

Embedding de "manzana":

[0.8, 0.1, 0.2, 0.5, ...]
  ↑    ↑    ↑    ↑
Comida Humano Joven Alegre
⚠️ Nota: En realidad las dimensiones NO tienen interpretaciones tan claras, pero capturan patrones útiles para la computadora

🎩 La "Magia" de los Embeddings

Operaciones famosas:

Rey - Hombre + Mujer ≈ Reina

París - Francia + México ≈ Ciudad de México

Programar - Código + Receta ≈ Cocinar

¿Por qué funciona?

  • → Embeddings capturan relaciones conceptuales
  • → "Realeza" es un concepto que se puede sumar/restar
  • → "Capital de" es una relación que se preserva
⚠️ No siempre funciona perfectamente, pero muestra el poder de las representaciones

Resumen de la Sesión
