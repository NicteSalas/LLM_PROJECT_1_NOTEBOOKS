Word Embeddings (Recap)

Ya lo conocemos:

"gato" → [0.2, 0.8, -0.3, ..., 0.5]  (300 dimensiones)
"perro" → [0.1, 0.7, -0.2, ..., 0.4]
"mesa"  → [-0.5, 0.1, 0.6, ..., -0.3]

✓ Ventajas:

  • Captura similitud entre palabras
  • Operaciones matemáticas (analogías)
  • Base sólida para NLP

✗ Limitación crítica:

NO captura el contexto de la oración
Cada palabra tiene UN solo embedding, sin importar dónde aparece

Sentence Embeddings

Idea: Representar frases completas como vectores

"El gato come pescado"  → [0.1, 0.4, 0.7, ..., 0.2]  (768D)
"El perro come carne"   → [0.1, 0.3, 0.8, ..., 0.3]
"El clima está soleado" → [-0.5, 0.2, 0.1, ..., -0.4]

✓ Ventajas:

  • Captura el significado de la frase COMPLETA
  • Frases similares → vectores cercanos
  • Ignora detalles sintácticos irrelevantes

📊 Aplicaciones:

  • Búsqueda semántica: "perro grande" encuentra "canino enorme"
  • Detección de paráfrasis: ¿Dicen lo mismo?
  • Clustering: Agrupar textos similares
  • Sistemas de recomendación: Contenido similar

Document Embeddings

Idea: Representar documentos completos

📄 Documento completo (cientos de palabras)
           ↓
    [0.3, -0.1, 0.8, ..., 0.5]
           ↓
     Vector único que representa
     todo el contenido semántico

✓ Modelos:

  • Bag-of-words: Tipo básico de document embedding
  • Doc2Vec: Extensión de word2vec
  • BERT [CLS]: Token especial para documento
  • Paragraph vectors: Aprendizaje no supervisado

📊 Aplicaciones:

  • Clasificación de documentos
  • Clustering de artículos
  • Detección de plagio
  • Búsqueda de documentos similares

El Problema del Context Embedding

Historia:

2013: Word2Vec → embeddings estáticos
2014: RNNs para procesar secuencias

Problema con RNNs:

Encoder RNN:
"I love llamas" → [📦 embedding único]
                       ↓
                  Decoder RNN:
              "Ik hou van lama's"
❌ Problema: Information Bottleneck
  • Todo comprimido en UN solo vector
  • Pierde información en oraciones largas
  • ¿Cómo representar un libro en 300 números?

💡 La Solución: Attention (2014)

El Breakthrough:

Idea: ¿Y si el decoder pudiera MIRAR diferentes partes del input según necesite?

Attention Mechanism:

  • No comprimir todo en un vector
  • El decoder "atiende" a diferentes palabras
  • Enfoque dinámico según contexto

Ejemplo:

Input:  "I     love    llamas"
Output: "Ik    hou     van     lama's"
         ↓      ↓       ↓        ↓
       atiende a diferentes partes del input

Cuando genera "lama's":
→ Atiende MUCHO a "llamas" (alta atención) ●
→ Atiende POCO a "I" (baja atención) ○

Visualización de Attention

Matriz de Atención:

I love llamas
Ik ● ○ ○
hou ○ ● ○
van ○ ● ○
lama's ○ ○ ●

● = Alta atención ○ = Baja atención

💡 Mensaje clave:
Attention permite al modelo ENFOCARSE en las partes relevantes

¿Cómo Funciona Attention?

Intuición (sin ecuaciones todavía):

Paso 1: Encoder procesa input

"I love llamas" → [h₁, h₂, h₃]
                   ↑   ↑   ↑
              Un vector por palabra

Paso 2: Decoder en cada paso

Decoder pregunta:
"¿Qué palabra del input es más relevante AHORA?"
    ↓
Calcula attention weights (importancia)
    ↓
Pondera los vectores del encoder
    ↓
Genera siguiente palabra

Paso 3: Resultado

Contexto dinámico en CADA paso
(no un solo embedding estático)

✓ Ventajas:

2. Interpretable

Debugging más fácil

3. Mejor performance

Self-Attention: La Magia ✨

Diferencia clave:

Attention en RNN:

Decoder atiende 
   al Encoder

Input → Encoder
         ↓
      Decoder ← atiende aquí

Self-Attention:

Una secuencia se 
atiende a SÍ MISMA

Input → Procesar
        ↓
     Atiende a sí mismo

Ejemplo:

"The animal didn't cross the street because it was too tired"
                                                   ↑
Self-attention permite que "it" atienda a "animal"
→ Entiende a qué se refiere "it"
💡 Beneficio clave:
• Captura relaciones dentro de la misma secuencia
• Procesa todo en paralelo (no secuencial)
• Puede "mirar" hacia adelante Y atrás simultáneamente

Encoder Block en Detalle

Multi-Head Self-Attention

Múltiples "cabezas" aprenden diferentes patrones

↓

Add & Norm

Residual connection + Layer Normalization

↓

Feed-Forward Network

Red neuronal simple (2 capas)

↓

Add & Norm

Otra residual connection + normalization

Se apilan N veces (típicamente 6-12 capas)

Decoder Block

Similar al encoder PERO con 3 componentes:

1. Masked Self-Attention

Solo ve palabras ANTERIORES
(no hace "trampa" viendo el futuro)

↓

2. Cross-Attention

Atiende al output del encoder
(como attention en RNNs)

↓

3. Feed-Forward Network

Procesa representaciones

Diferencia clave con encoder:
Decoder tiene attention layer ADICIONAL que atiende al encoder

¿Por Qué Funcionó Tan Bien?

Ventajas que revolucionaron el campo:

✓ 1. Paralelización

RNNs: procesan palabra por palabra (lento)
Transformers: procesan todas a la vez (rápido)

✓ 2. Long-range dependencies

Attention directa entre cualquier par de tokens
No importa qué tan lejos estén

✓ 3. Escalabilidad

Funciona MEJOR con más datos y más parámetros
(¡No se "satura"!)

2017: Transformer paper
2018: BERT (encoder-only)
2019: GPT-2 (decoder-only)
2020: GPT-3 (175B parámetros)
2023: ChatGPT 🚀

[OPCIONAL] La Matemática Básica

Self-Attention en 3 pasos:

Paso 1: Crear Query (Q), Key (K), Value (V)

Q = X · Wq
K = X · Wk  
V = X · Wv

donde X es la entrada y W son matrices aprendidas

Paso 2: Calcular Attention Weights

Attention = softmax(Q·K^T / √d_k)

(d_k = dimensión, para estabilizar valores)

Paso 3: Aplicar a Values

Output = Attention · V
No necesitan memorizar esto, solo entender que:
• Es álgebra lineal + softmax
• Se puede calcular eficientemente
• Se aprende durante entrenamiento
